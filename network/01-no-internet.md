## Нет интернета / Нет подключения к сети

# Симптомы:
- Нет значка Wi-Fi / Ethernet
- IP-адрес 169.254.x.x  
- Сайты не открываются

# Пошаговая диагностика:

**1. Проверь физическое подключение**
   - Вставлен ли кабель? Wi-Fi включён?

**2. Windows:**
   
   Открываем командную строку (cmd) и используем следующие команды для диагностики сети:
   
   `ipconfig /all` - Полная конфигурация TCP/IP для всех сетевых адаптеров. Отображает IPv4 и IPv6-адреса, маски подсети, шлюзы по умолчанию, MAC-адреса адаптеров, информацию о DNS-серверах.
   
   `ipconfig /release` - Отправляет сообщение DHCPRELEASE серверу DHCP, чтобы освободить текущую конфигурацию DHCP и отменить конфигурацию IP-адреса. Это отключает TCP/IP для адаптеров, настроенных для автоматического получения IP-адресов
   
   `ipconfig /renew` - Запрашивает у DHCP-сервера новый IP-адрес для всех адаптеров или для указанного адаптера. Обычно используется после команды ipconfig /release для восстановления подключения.
   
   `ipconfig /flushdns` - Очищает кэш сопоставителя DNS-клиента, удаляя сохранённые записи DNS. кЭто может помочь при проблемах с доступом к сайтам после смены DNS-серверов или если сайт поменял IP-адрес, а компьютер продолжает использовать старый.
   
**3. Linux:**

   Команды для терминала:
   
     ip addr show (сокр. ip a) -  отображает информацию о сетевых интерфейсах, их IP-адресах и свойствах.
   
     sudo dhclient -r && sudo dhclient:
       (sudo dhclient -r — освобождает текущую аренду DHCP-адреса. То есть отменяет временное присвоение IP-адреса от DHCP-сервера.
         sudo dhclient — запрашивает новый IP-адрес у DHCP-сервера для текущего сетевого интерфейса. Если указать имя интерфейса, команда обновит аренду IP-адреса для указанного интерфейса.)
