# Windows Update не устанавливается / зависает (Windows 11/10)

**Симптомы:**  
- Ошибки 0x80070002, 0x80070643, 0x8024402f  
- Обновления висят на 0% или «Не удалось установить»

**Пошаговая диагностика:**

1. Запусти встроенный troubleshooter:
   Параметры -> Система -> Устранение неполадок -> Другие средства устранения неполадок -> Центр обновления Windows

2. Очисти кэш обновлений (в PowerShell от имени администратора):

   `net stop wuauserv` - останавливает службу «Центр обновления Windows».
   `net stop cryptSvc` - останавливает службу криптографии, которая отвечает за проверку цифровых подписей и сертификатов.
   `net stop bits` - останавливает службу BITS. Она отвечает за фоновую передачу данных, в т. ч. загрузку обновлений Windows.
   `net stop msiserver` - останавливает службу установщика Windows, которая управляет установкой и удалением программ.
   `ren C:\Windows\SoftwareDistribution SoftwareDistribution.old` - переименовывает папку SoftwareDistribution. В ней хранятся временные файлы загруженных обновлений. Переименование позволяет системе создать новую чистую папку при следующем запуске службы.
   `ren C:\Windows\System32\catroot2 Catroot2.old` - переименовывает папку Catroot2. В ней содержатся подписи пакетов обновлений. Переименование помогает устранить ошибки, связанные с повреждёнными подписями.
   `net start wuauserv` - запускает службу «Центр обновления Windows».
   `net start cryptSvc` - запускает службу криптографии.
   `net start bits` - запускает службу BITS.
   `net start msiserver` - запускает службу установщика Windows.

3. Запустить SFC (System File Checker) или DISM (Deployment Image Servicing and Management) для проверки целостности системных файлов:
   `sfc /scannow`

4. После очистки перезагрузить компьютер и попробовать обновиться снова.
