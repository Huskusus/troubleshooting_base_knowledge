# Сервис не запускается (systemctl)

**Симптомы:**  
- Сервис в статусе `failed` / `inactive (dead)`  
- `systemctl status nginx` показывает ошибки

**Пошаговая диагностика:**

1. Проверить статус:
   
   `systemctl status <service-name>` - проверяет статус указанной службы.
   `journalctl -xeu <service-name>` -  запрашивает и отображает логи для указанной службы с дополнительными опциями (в нашем случае -x добавляет пояснительные тексты из каталога сообщений к логам, -u — указывает, что данные должны быть отображены для конкретного юнита

2. Перезапустить сервис:

   `sudo systemctl restart <service-name>`
   `sudo systemctl enable <service-name>`

3. Проверить логи:

   `journalctl -u <service-name>`

4. Перезагрузка сервера как последний шаг
