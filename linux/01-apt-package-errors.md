# Ошибки при установке пакетов (apt / dpkg)

**Симптомы:**  
- `E: Unable to locate package ...`  
- `E: Could not get lock /var/lib/dpkg/lock`  
- `apt update` падает с ошибками

**Пошаговая диагностика:**

1. Обновить списки пакетов:
   `sudo apt update`
   
2. Исправить ошибки блокировки:
   `sudo rm /var/lib/dpkg/lock-frontend` - Удаляет файл блокировки интерфейса dpkg. Этот файл создаётся, когда какой‑либо процесс (например, apt или графический менеджер пакетов) использует систему управления пакетами.
   `sudo rm /var/cache/apt/archives/lock` - Удаляет файл блокировки кэша пакетов APT. Этот файл блокирует доступ к каталогу с кэшированными .deb-пакетами (/var/cache/apt/archives/).
   `sudo dpkg --configure -a` - Настраивает все распакованные, но не полностью установленные пакеты. Команда проверяет состояние пакетов и пытается завершить их установку.

3. Очистить кэш и обновить:
   `sudo apt clean`
   `sudo apt autoclean`
   `sudo apt autoremove`
   `sudo apt update`
